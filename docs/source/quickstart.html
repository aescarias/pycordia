Note

Note that Pycordia works best on Python 3.8+, although 3.7 is supported.

Pycordia can be installed in two possible ways.

Migrating from Discord.py (or its forks)Â¶

Pycordia is not a drop-in replacement for Discord.py, Pycordia has been built from scratch and shares a different codebase than the one from Discord.py. Please note this before using our library. Itâs up to you to decide whether learning a different codebase is worth it for your current bot. Pycordia is still excellent if developing new bots.

Virtual EnvironmentsÂ¶

We recommend using Pycordia inside a virtual environment to not conflict with other installed packages.

The following shows steps on how to create and activate one by using venv:

$ python3 -m venv venv
$ source venv/bin/activate

$ py -3 -m venv venv
$ venv/Scripts/activate

Installing from PyPi (recommended)Â¶

The easiest way to install Pycordia is from PyPi through pip.

$ python3 -m pip install pycordia

$ py -3 -m pip install pycordia

Installing from GitÂ¶

If you prefer to stay on the edge and install Pycordia directly from Git, proceed with the following:

$ python3 -m pip install git+https://github.com/angelcarias/pycordia.git

$ py -3 -m pip install git+https://github.com/angelcarias/pycordia.git

ExampleÂ¶

Verify Pycordia is properly installed by doing:

import pycordia

Try creating your first bot. We assume you have setup a Discord application and have obtained a bot token.

import pycordia
from pycordia import models
import dotenv
import os

dotenv.load_dotenv()

client = pycordia.Client(intents=pycordia.Intents.all())

@client.event
async def on_ready(event):
   print(f"{client.user} active")

client.run(os.getenv("DISCORD_TOKEN"))

This example also assumes youâre using python-dotenv or a .env file loader. We recommend you use environment variables in .env files to run a lower risk of your Discord bot token leaking.
